## 3. Surprises

1. **D2 false-blocker on PyPI publishing.** The agent flagged `gecko-mcp==0.1.4` as not-on-PyPI; verification took 30s and showed it was actually published. Encoding: when an agent flags "blocked on external-system publishing," verify with a 5-second `curl https://pypi.org/pypi/<pkg>/json` before accepting the blocker.
2. **5 tests polluted by `.env`.** R1's secret-presence tests inadvertently read the dev `.env`, masking failures locally that would surface in CI. Caught and fixed in commit `1aa7657`. Encoding: any test that asserts a config default must use `_env_file=None` at construction or explicit setenv — `monkeypatch.delenv` alone is insufficient because `pydantic-settings` reads `.env` in addition to process env.
3. **Mongo cutover was anticlimactic.** The risk register treated M4 hybrid parity as the long pole. In practice, `$vectorSearch` + `$search` + app-side RRF matched Postgres `match_chunks_hybrid` ordering within ±1 on first run because there were *no historical chunks* — the fresh-corpus framing (`project_mongo_cutover_no_backfill`) eliminated the parity surface entirely. M4 came in a half-day under estimate.
4. **R2 batching gap.** Per-question reranker calls compound on multi-question synthesis (5 calls = ~4s added). R2 shipped per-question; per-session batching is a TODO carrying into S20.
5. **S2 found a sleeper.** Four V1 source ids are hardcoded across 5 prompt versions with no consistency test. This is **Pattern A (parallel `Literal` redeclarations)** repeating in prompt-string land. Bigger S20 risk than the v1_block tear-out itself.
6. **D3 surfaced a verdict-parse drift.** All 6 regressions vs S17 are `actual_verdict=unknown` (parse failures), not polarity flips. The judge prose has shifted slightly post-cutover such that `extract_verdict()` no longer matches. R3 inherits this in both arms; delta is still measurable but baseline is noisier than ideal. Encoding: when a parse step lives between LLM output and verdict accounting, a regression test on the *parse* is as important as on the *verdict* — bug class is "model output drifted under our format expectations."

## 4. Wedge defensibility status

**Current sentence (locked in S18-D2):**
> Gecko produces grounded, adversarial verdicts on pre-ideas — a judgment you can buy, sell, or stake on.

**Verdict: still right, but the demo screenshots and S1 surface spec exposed a sharper sub-claim.**

The "grounded, adversarial verdicts" half remains the moat. Voyage rerank (when R3 trips) hardens "grounded." The S17 5-voice debate hardens "adversarial."

The "buy, sell, or stake on" half is now backed by an actual artifact — the **verdict hash → shareable URL → x402-paywalled detail view** chain specified in S1. That's defensible *because* the upstream artifact (a corpus-anchored, dissent-preserving verdict) is hard to replicate, not because the URL wrapper is novel.

**Pattern D check:** is "tradeable judgment" defensible against Perplexity/ChatGPT/Bazaar? The wrapper is not — anyone can paywall a URL. The *contents* are: a sha256-stable, citation-grounded verdict over a heterogeneous-provider corpus where dissent is a first-class output. Lead with the artifact, not the marketplace.

**No reframe needed for S20.** Hold the S18 sentence; let S20 web3-eng make the URL/settlement layer real so the second half stops being aspirational.
