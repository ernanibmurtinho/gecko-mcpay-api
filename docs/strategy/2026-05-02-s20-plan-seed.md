## Candidate tickets (ordered by leverage)

### Lead with these three

1. **S20-V1BLOCK-CONSISTENCY-01** — `ai-ml`, **M**. Add a Pattern-A-style consistency test for the four V1 source ids hardcoded across 5 prompt versions (surfaced by S19-S2). One canonical Python module, schema-drift test, gates the v1_block tear-out. *Rationale: this is the same class of bug as `PaymentMode` pre-S12.5; tear-out without it will produce silent prompt-output divergence.* **(Lead 1.)**
2. **S20-COHERENCE-VERDICT-LABEL-01** — `ai-ml`, **M**. Add a `KILL` (or `NO`) verdict label for premise-incoherent ideas, triggered when ≥2 critics flag incoherence. Update PRD/skill.md + verdict-hash inputs. *Rationale: stress-matrix idea #4 exposes a real failure mode — politely REFINE-ing nonsense ideas erodes trust faster than any other signal loss.* **(Lead 2.)**
3. **S20-RERANKER-BATCH-01** — `software-eng`, **S**. Per-session reranker batching (R2 TODO). One Voyage call per session, not per question. Latency budget reclaim = ~2.5–3s on 5-question synthesis. *Rationale: directly protects the <90s demo runbook target as we add features; R2 already flagged this.* **(Lead 3.)**

### Next tier — protective + carry-forward

4. **S20-PROVIDER-MIX-FLOOR-01** — `ai-ml`, **S**. Soft-cap any single `provider_kind` at ~50% of citation set; require ≥3 distinct kinds. Surface a `provider_mix_flag` in verdict metadata. *Rationale: stress-matrix #1 (crypto) will skew twit.sh-heavy without this.*
5. **S20-DISTRIBUTION-CRITIC-01** — `ai-ml`, **S**. New critic voice in 5-agent debate focused on GTM/distribution, gated on B2B / 2-sided idea classification. *Rationale: stress-matrix #3 (hotels) — model nails product-market fit and misses the distribution moat.*
6. **S20-SELF-CITATION-GUARD-01** — `ai-ml` + `data-eng`, **S**. Down-weight chunks resolving to `app.geckovision.tech` / our own repos when idea topic overlaps Gecko's product space. *Rationale: stress-matrix #5 (dogfood-on-itself) — circular citations look like signal but aren't.*
7. **S20-FEATURE-NOT-PRODUCT-CRITIC-01** — `ai-ml`, **S**. Adversarial critic prompt extension for `crowded-but-differentiable` and `mature` gap classifications. *Rationale: stress-matrix #2 (API gateway) — confident GO on what's actually a feature.*
8. **S20-VERDICT-PARSE-REGRESSION-01** — `ai-ml`, **XS**. D3 surfaced 6 ideas where `actual_verdict=unknown` (parse failure, not polarity flip). Add a regression test that locks the judge-prose → verdict-token mapping; tighten `extract_verdict` heuristics. *Rationale: parse drift is silent and corrupts every downstream metric; cheap to defend against.*
9. **S20-SUPABASE-CHUNK-DROP-01** — `data-eng`, **XS**. Carry-forward of S19-S3 if soak window expired clean. Drop `chunks` + `chunk_embedding_cache` Supabase tables. *Rationale: storage hygiene; locks in the cutover.*
10. **S20-VOYAGE-MODEL-SWAP-01** — `data-eng` + `ai-ml`, **L**. `text-embedding-3-small` → `voyage-3` (or `voyage-context-3`). Re-embed entire Mongo corpus. *Rationale: deferred from S19 per `project_voyage_ai_s19_followup`. Run only if R3 trips and rerank wins are real — model swap on top of a good reranker compounds; on top of a bad one wastes spend.*

## What I'd unilaterally cut as scope creep

**S20-VOYAGE-MODEL-SWAP-01** if R3 hasn't tripped the gate. Re-embedding the entire corpus to swap embedding models when we don't yet have evidence that Voyage's reranker beats no-rerank is exactly the kind of "do the next shiny thing" creep that Pattern B warns about. Reranker first, model swap second — and only if reranker wins. If R3 reverts, this ticket dies.
