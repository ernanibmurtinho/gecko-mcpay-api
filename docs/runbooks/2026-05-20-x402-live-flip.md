## 3. SSM SecureString env wiring

These four SSM parameters MUST exist before the deploy. Operator (founder) creates them in `us-east-2`; this runbook does not — secrets never flow through Claude. The ECS task definition references them via `secrets:` (not `environment:`) so values never appear in task descriptions.

| SSM Parameter Name | ECS env var | Type | Notes |
|---|---|---|---|
| `/gecko/prod/buyer_wallet_private_key_path` | `BUYER_WALLET_PRIVATE_KEY_PATH` | `SecureString` | Path *inside* the container where the keypair is mounted. Recommend `/secrets/buyer.json`. The key file itself is delivered via the existing secrets-volume init container, NOT bundled in the image. |
| `/gecko/prod/buyer_wallet_pubkey` | `BUYER_WALLET_PUBKEY` | `String` | Base58 Solana pubkey of the buyer. Not secret; kept in SSM for env consistency. |
| `/gecko/prod/x402_live_tools` | `GECKO_X402_LIVE_TOOLS` | `String` | Value: `gecko_trade_research,gecko_trade_research_pro`. NOT `gecko_research`. |
| `/gecko/prod/x402_kill_switch` | `GECKO_X402_KILL_SWITCH` | `String` | Default `false`. Toggling to `true` is the rollback lever (§6). |

**ECS task definition `secrets:` block (excerpt):**

```yaml
ContainerDefinitions:
  - Name: gecko-api
    Secrets:
      - Name: BUYER_WALLET_PRIVATE_KEY_PATH
        ValueFrom: !Sub arn:aws:ssm:${AWS::Region}:${AWS::AccountId}:parameter/gecko/prod/buyer_wallet_private_key_path
      - Name: BUYER_WALLET_PUBKEY
        ValueFrom: !Sub arn:aws:ssm:${AWS::Region}:${AWS::AccountId}:parameter/gecko/prod/buyer_wallet_pubkey
      - Name: GECKO_X402_LIVE_TOOLS
        ValueFrom: !Sub arn:aws:ssm:${AWS::Region}:${AWS::AccountId}:parameter/gecko/prod/x402_live_tools
      - Name: GECKO_X402_KILL_SWITCH
        ValueFrom: !Sub arn:aws:ssm:${AWS::Region}:${AWS::AccountId}:parameter/gecko/prod/x402_kill_switch
    Environment:
      - Name: X402_MODE
        Value: live
      - Name: X402_NETWORK
        Value: solana-mainnet
      - Name: X402_SELLER_PAYTO
        ValueFrom: !Sub arn:aws:ssm:${AWS::Region}:${AWS::AccountId}:parameter/gecko/prod/seller_pay_to
```

> Founder responsibility — set the SSM values BEFORE running the deploy. This runbook does not run `aws ssm put-parameter` commands; do those manually (per `feedback_no_secret_access`).

## 4. Deploy command

After §1 checklist green and §3 SSM params populated:

```bash
./infra/deploy.sh --cert arn:aws:acm:us-east-2:<ACCOUNT>:certificate/<EXISTING-CERT-ARN>
```

The script will:
1. Build the Docker image from the current working tree.
2. Push to ECR.
3. Update the CloudFormation stack with the new task-definition env.
4. Force ECS service update; min-tasks=2 ensures no downtime.

**Expected duration:** ~6 min image build + ~4 min ECS rollout = ~10 min.

**Stop signs (abort + roll back):**
- Image build fails → fix locally, restart.
- CloudFormation stack rollback triggered → leave it rolled back, post-mortem, do NOT retry without diagnosing.
- `/health` returns non-200 within 5 min of rollout → §6 rollback immediately.

## 7. Rollback procedure

**One command, blast time ≤ 60 seconds (env change only, no redeploy):**

```bash
aws ssm put-parameter \
  --region us-east-2 \
  --name /gecko/prod/x402_kill_switch \
  --value true \
  --type String \
  --overwrite
```

The `assert_live_settlement_allowed` gate guard
(`packages/gecko-core/src/gecko_core/payments/__init__.py`) reads
`GECKO_X402_KILL_SWITCH` per request — flipping the SSM value flips
behaviour on the next request without restarting the task. ECS does
NOT cache SSM SecureString values across requests for this var (it's
mounted via `secrets:` at task start, but the guard re-reads env on
every dispatch — verified in unit tests).

Blast time = SSM propagation (~30s) + first-request fall-through.

**Confirm degraded mode after the flip:**

```bash
curl -s https://api.geckovision.tech/healthz | jq '.x402'
```

Expected after kill-switch flip:
```json
{
  "mode": "live",
  "kill_switch": true,
  "effective_mode": "stub",
  "live_blocked_recent": ">0"
}
```

If `effective_mode` still reports `live` after 60s, the guard is not re-reading env per request — escalate to the CloudFormation rollback below immediately.

**If the kill-switch flip is insufficient (e.g. the guard itself is broken):**

```bash
aws cloudformation update-stack \
  --region us-east-2 \
  --stack-name gecko-api \
  --use-previous-template \
  --parameters \
    ParameterKey=X402Mode,ParameterValue=stub \
    ParameterKey=X402Network,ParameterValue=solana-devnet \
    ParameterKey=X402LiveTools,ParameterValue="" \
  --capabilities CAPABILITY_NAMED_IAM
```

Then `./infra/deploy.sh --cert <ARN>` to force task rotation. Blast time ~6 min.
